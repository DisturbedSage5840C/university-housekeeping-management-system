1. Project Overview

ILGC Tracker is an AI-powered Campus Housekeeping & Hygiene Compliance Tracker designed to solve a persistent problem on campus: the failure to refill essential hygiene supplies (tissues, soaps, sanitizers) for days at a time, and the lack of consistent monitoring of housekeeping standards. The system gives residents an instant way to report shortages and hygiene issues, uses machine learning to auto-categorize and prioritize those complaints, and gives administrators real-time visibility into compliance gaps — ensuring timely replenishment and accountability.

The Problem: Recurring housekeeping lapses on campus — essential hygiene supplies (tissues, soaps) go unrefilled for up to 4 consecutive days. There is no structured system for residents to report shortages, no tracking of replenishment schedules, and no accountability mechanism for housekeeping staff. This lapse in basic sanitation resources poses health and hygiene risks campus-wide.
The Solution — ILGC Tracker: Home-first website entry with Supervisor/Admin sign-in and role-aware routing • Building/Floor/Direction-wise washroom data tracking • Supply inventory monitoring (soap, tissue, sanitizer levels) • AI complaint analysis with 12 categories (including hygiene, supplies, cleaning) • Priority prediction (hygiene issues auto-flagged high/urgent) • 2-way work verification backend pipeline (LLM image analysis + supervisor approval endpoint) • False reporting detection • Supervisor complaints overview + supply stats updater + Open Staff Tools access • Restocking timeline tracking • NLP sentiment analysis • AI auto-categorization with review flow • Complaint detail popup with full AI breakdown • Image upload for photographic evidence • Predictive maintenance to anticipate supply shortages • Light/Dark theme • Real-time dashboard for monitoring compliance • Mobile app shell for on-the-go access

How It Works (Simple)

  1. Student/Faculty notices that soap dispensers in the H2 – 2nd Floor South Wing washroom have been empty for 2 days. They open ILGC Tracker and submit a complaint: "No soap in H2 2nd floor south wing washroom for 2 days, tissue dispensers also empty" — attaches a photo as evidence, lets AI auto-detect the category
  2. AI automatically categorizes it as "hygiene" (with confidence scores across all 12 categories), predicts "high" priority (hygiene shortages get elevated priority), analyzes negative sentiment, detects urgency keywords ("empty", "2 days"), extracts entities ("soap", "tissue", "H2 2nd floor south wing"), and suggests actions ("Restock soap and tissue dispensers immediately")
  3. Student reviews AI-detected category + scores in a picker, confirms or changes, then submits
  4. Admin/Supervisor sees the complaint on the dashboard with AI tags (category, sentiment, urgency, priority). Clicks the card → detail popup shows resident info, room/location, attached photo evidence, full AI analysis with confidence bars, suggested actions, estimated resolution time. Admin can escalate false reports to supervisors; supervisors assign complaints to their own staff
  5. Cleaning Staff receives the task, restocks supplies, uploads proof photo. LLM verifies the uploaded image for cleanliness/completion. Supervisor reviews and approves the submission — 2-way verification ensures accountability
  6. System tracks supply levels building-wise (soap, tissue, sanitizer), sends consumption-based reminders to supervisors, detects false reporting patterns, logs everything with timestamps — admin can identify patterns (e.g., which buildings/floors have repeated shortages, restocking timelines) to improve scheduling

13. Deployment Guide

Prerequisites

  • Docker & Docker Compose V2
  • macOS, Linux, or Windows with WSL2

One-Command Deploy

cd "/Users/maruteymani/Documents/ILGC Tracker"
chmod +x deploy.sh
./deploy.sh

What deploy.sh Does (step by step)

  1. Checks Docker & Docker Compose installed
  2. Creates .env from template with auto-generated secrets (JWT, DB password, Redis password)
  3. Generates self-signed SSL certificates
  4. Copies index.html → frontend-dist/
  5. Builds all Docker images (docker compose build --no-cache)
  6. Starts 6 containers (docker compose up -d)
  7. Waits for PostgreSQL to be ready
  8. Runs database migrations and seeds
